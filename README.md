# E-Commerce System
## Project Overview
This project is a basic e-commerce system built using NestJS and MongoDB. It is designed to demonstrate backend development skills by implementing a system where:

- Unauthenticated users can view approved products.
- Authenticated users can manage their own products.
- Admins can manage both users and products.
The project follows a modular structure, utilizing NestJS's powerful dependency injection system, and leverages MongoDB for data storage.

## Objectives
- Unauthenticated Users: Can view a list of approved products.
- Authenticated Users: Can create, update, and delete their own products.
- Admin Users: Can manage all users and products, including approving products for public viewing.

## Project Structure
The project is organized into the following modules:

- **Auth Module:** Handles authentication and authorization.
- **Users Module:** Manages user data and roles.
- **Products Module:** Manages product data, including CRUD operations and approval status.
## Setup Instructions
### Prerequisites
Before setting up the project, ensure you have the following installed:

- **Node.js** (version 14 or higher)
- **npm** (version 6 or higher)
- **MongoDB** (either locally installed or using a service like MongoDB Atlas)
## Installation
### Clone the repository:
```
git clone https://github.com/your-username/ecommerce-system.git
cd ecommerce-system
```

### Install dependencies:
```
npm install
```

### Configure Environment Variables:

Create a .env file in the root directory and add the following environment variables:
```
MONGODB_URI=mongodb://localhost/ecommerce
JWT_SECRET=your_jwt_secret
PORT=3000
```

- **MONGODB_URI**: The connection string for your MongoDB instance.
- **JWT_SECRET:** A secret key for signing JSON Web Tokens.
- **PORT:** The port number for the application (default is 3000).
  
## Running the Application Locally
### Start the MongoDB server:

If MongoDB is installed locally, you can start it with:
```
mongod
```
If you're using MongoDB Atlas, ensure your MONGODB_URI is correctly set in the .env file.

## Run the NestJS application:
```
npm run start:dev
```
The application will start on the port defined in your .env file (default is http://localhost:3000).

## Access the API Documentation:
The API documentation is automatically generated by Swagger and can be accessed at:
```
http://localhost:3000/api
```
## Running Tests
The project includes unit tests to ensure the functionality of the services and controllers.

### Run tests:
```
npm run test
```
Run tests with coverage:
```
npm run test:cov
```
